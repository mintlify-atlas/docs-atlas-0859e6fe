The Discord OTP Bot uses intelligent pattern matching to detect OTP codes across various email formats and services. This page explains the regex patterns used and how the extraction works.

## Pattern Matching Logic

The bot searches both the email subject and body using multiple regex patterns in priority order. The first matching pattern returns the OTP code.

### Pattern Priority

Patterns are checked in the following order (from `otp_bot.py:40-60`):

```python
patterns = [
    # Instacart style: "147477 is your instacart verification code"
    r'(\d{6})\s+is\s+your\s+[\w\s]+\s+verification\s+code',
    
    # Bolded codes like **147477**
    r'\*\*(\d{4,8})\*\*',
    
    # Standalone 4-8 digit codes
    r'\s(\d{4,8})\s',
    
    # Common OTP phrases
    r'code:?\s*[:\-]?\s*(\d{4,8})',
    r'verification\s+code:?\s*[:\-]?\s*(\d{4,8})',
    r'OTP:?\s*[:\-]?\s*(\d{4,8})',
    r'one.time.*?code:?\s*[:\-]?\s*(\d{4,8})',
    r'your.*?code.*?is:?\s*[:\-]?\s*(\d{4,8})',
    
    # Fallback: any 4-8 digit number
    r'\b\d{4,8}\b'
]
```

<Note>
Patterns use case-insensitive matching (`re.IGNORECASE`) to handle variations in capitalization.
</Note>

## How Pattern Matching Works

The extraction process follows these steps:

<Steps>
  <Step title="Combine Email Parts">
    The bot concatenates the email subject and body text into a single search string:
    ```python
    search_text = f"{email_subject} {email_text}"
    ```
  </Step>
  
  <Step title="Apply Patterns Sequentially">
    Each regex pattern is tested against the search text in priority order until a match is found.
  </Step>
  
  <Step title="Return First Match">
    The first pattern that successfully matches returns the extracted OTP code:
    ```python
    for pattern in patterns:
        matches = re.findall(pattern, search_text, re.IGNORECASE)
        if matches:
            print(f"🔑 OTP found with pattern: {matches[0]}")
            return matches[0]
    ```
  </Step>
  
  <Step title="Fallback to Generic">
    If no specific pattern matches, the fallback pattern (`\b\d{4,8}\b`) catches any 4-8 digit number.
  </Step>
</Steps>

## Regex Pattern Breakdown

Let's break down the key patterns:

### Verification Code Pattern

```regex
verification\s+code:?\s*[:\-]?\s*(\d{4,8})
```

- `verification` - Matches the word "verification"
- `\s+` - Matches one or more whitespace characters
- `code` - Matches the word "code"
- `:?` - Optionally matches a colon
- `\s*` - Matches zero or more whitespace
- `[:\-]?` - Optionally matches `:` or `-`
- `(\d{4,8})` - **Captures 4-8 digits** (the OTP)

### Bolded Code Pattern

```regex
\*\*(\d{4,8})\*\*
```

- `\*\*` - Matches two asterisks (escaped)
- `(\d{4,8})` - **Captures 4-8 digits** (the OTP)
- `\*\*` - Matches two closing asterisks

### Flexible Code Pattern

```regex
code:?\s*[:\-]?\s*(\d{4,8})
```

- `code` - Matches the word "code"
- `:?` - Optionally matches a colon
- `\s*` - Matches zero or more whitespace
- `[:\-]?` - Optionally matches `:` or `-`
- `(\d{4,8})` - **Captures 4-8 digits** (the OTP)

<Info>
All capture groups use `(\d{4,8})` to match OTP codes between 4 and 8 digits, which covers the vast majority of verification codes.
</Info>

## Limitations

<Warning>
The bot may struggle with:
- **Alphanumeric codes** (e.g., `A1B2C3`) - only numeric codes are supported
- **Codes outside 4-8 digits** - codes like `123` or `123456789` won't match
- **Image-based OTPs** - codes embedded in images cannot be extracted
</Warning>

## Adding Custom Patterns

If you need to support a specific OTP format not covered by the default patterns, you can modify the `patterns` list in `otp_bot.py:41-60`:

```python
patterns = [
    # Add your custom pattern at the top for priority
    r'your_custom_pattern_here',
    
    # Existing patterns...
    r'(\d{6})\s+is\s+your\s+[\w\s]+\s+verification\s+code',
    # ...
]
```

<Tip>
Place more specific patterns at the top of the list to ensure they're checked before generic fallback patterns.
</Tip>
